Una guía visual para entender cómo diseñamos software moderno.

El Proceso de Desarrollo

Un proceso es el camino que seguimos para construir software de calidad.

  • Cascada: Pasos fijos uno tras otro.
  • Iterativo: Ciclos que repiten el proceso para mejorar el producto.
Analogía: Construir una casa es Cascada (no puedes poner el techo sin las paredes). Pintar un cuadro es Iterativo (haces un boceto y vas agregando capas de detalle).

Análisis vs. Diseño

Antes de programar, debemos entender el problema y planear la solución.

Análisis: Se enfoca en el qué. Entender los requisitos del cliente.
Diseño: Se enfoca en el cómo. Crear el plano técnico de la solución.
Analogía: Cuando tienes hambre y decides comer algo nutritivo, haces Análisis. Cuando eliges una receta específica y preparas los ingredientes, haces Diseño.

Objetos y Clases

La base de este paradigma es organizar el código en "objetos".

Objeto: Una entidad con estado (datos) y comportamiento (acciones).

Analogía: La receta para fabricar un coche es la Clase. El coche real que conduces (con su color y motor) es el Objeto.
